# איך מעלים את האתר לאינטרנט – מדריך פשוט, שלב אחר שלב

המדריך הזה מניח שאתה לא מבכיר בתחום. כל שלב הוא פעולה אחת. בסוף יהיה לך אתר חי
בכתובת אינטרנט אמיתית, חינם, והנתונים (המתחמים, התמונות) יישמרו לתמיד.

**איך זה עובד בקצרה:** הקוד יושב ב-GitHub (כמו "ענן" לקבצים). אתר בשם Render לוקח
את הקוד משם ומריץ אותו באינטרנט. כשמוסיפים מתחם דרך אזור הניהול, הנתונים נשמרים
בחזרה ל-GitHub אוטומטית. הכל חינם.

---

## חלק א' – פתיחת חשבון GitHub והעלאת הקוד

### 1. פתיחת חשבון GitHub
- כנס ל-https://github.com ולחץ **Sign up**. הירשם עם אימייל וסיסמה. (חינם)

### 2. יצירת מאגר (repository)
- אחרי ההרשמה, לחץ על הכפתור **+** למעלה מימין ► **New repository**.
- ב-**Repository name** כתוב: `kal-nofesh`
- בחר **Private** (פרטי).
- לחץ **Create repository**.

### 3. העלאת הקבצים
הדרך הכי פשוטה בלי להתקין כלום:
- בעמוד המאגר החדש, לחץ על הקישור **uploading an existing file**.
- גרור לשם את **כל** הקבצים והתיקיות מתוך תיקיית `קל-נופש`
  (חוץ מהתיקייה `node_modules` אם היא קיימת – אותה לא צריך).
- למטה לחץ **Commit changes**.

> אם זה מסובך לך – תגיד לי ואני אכין לך את פקודות ההעלאה המדויקות.

### 4. יצירת "מפתח" שיאפשר לאתר לשמור נתונים
- כנס ל-https://github.com/settings/tokens
- לחץ **Generate new token** ► **Generate new token (classic)**.
- ב-**Note** כתוב: `kal-nofesh`
- ב-**Expiration** בחר **No expiration**.
- סמן את התיבה **repo** (הראשונה למעלה – היא מסמנת את כל מה שמתחתיה).
- למטה לחץ **Generate token**.
- **העתק את המפתח שמופיע ושמור אותו בצד** (משהו כמו `ghp_xxxxxxxx`). לא תוכל לראות
  אותו שוב אחרי שתעזוב את הדף.

## חלק ב' – העלאת האתר לאוויר עם Render

### 5. פתיחת חשבון Render
- כנס ל-https://render.com ולחץ **Get Started** / **Sign up**.
- הירשם **עם חשבון GitHub** (כפתור "GitHub") – זה מחבר ביניהם אוטומטית.

### 6. יצירת האתר
- בלוח של Render לחץ **New +** ► **Blueprint**.
- בחר את המאגר `kal-nofesh`. Render יזהה לבד את קובץ ההגדרות (`render.yaml`).
- לחץ **Apply**.

### 7. הזנת ההגדרות הסודיות
Render יבקש למלא כמה ערכים (Environment Variables). מלא:

| שם | מה לכתוב |
|----|----------|
| `ADMIN_PASSWORD` | סיסמה שתבחר לכניסה לאזור הניהול (תזכור אותה!) |
| `GITHUB_TOKEN` | המפתח שהעתקת בשלב 4 (`ghp_...`) |
| `GITHUB_REPO` | שם המשתמש שלך ב-GitHub ואז `/kal-nofesh`. לדוגמה: `netanel/kal-nofesh` |

(`SESSION_SECRET` נוצר אוטומטית – לא צריך לגעת.)

- לחץ **Apply / Create**.

### 8. זהו – האתר באוויר
- אחרי 1-2 דקות תקבל כתובת כמו `https://kal-nofesh.onrender.com`.
- זאת הכתובת של האתר שלך. אפשר לפתוח אותה בדפדפן ולשלוח לאנשים.

> **הערה אחת:** בגרסה החינמית, אם אף אחד לא נכנס לאתר במשך זמן מה, הכניסה הבאה
> עלולה לקחת חצי דקה לטעון ("האתר מתעורר"). זה נורמלי. אם בעתיד תרצה שזה יהיה
> מיידי תמיד, אפשר לשדרג בלחיצה אחת (~25 ש"ח לחודש) – תגיד לי ואסביר.

## חלק ד' – קניית דומיין ושינוי הכתובת (לכשתרצה)

הדומיין הזמני (`kal-nofesh.onrender.com`) עובד מצוין להתחלה. כשתרצה כתובת משלך
(כמו `kal-nofesh.co.il`):

1. בלוח Render, כנס לאתר שלך ► **Settings** ► **Custom Domains** ► **Add Custom Domain**.
2. הקלד את הדומיין שתרצה. Render יראה לך אם הוא פנוי ויאפשר לקנות אותו (או שתקנה
   אצל רשם דומיינים אחר ותחבר).
3. Render ידאג לאבטחה (HTTPS) אוטומטית.
4. מאותו רגע האתר יענה גם בכתובת החדשה.

> כשתגיע לשלב הזה – תגיד לי ואלווה אותך צעד-צעד. שינוי דומיין הוא רק הוספת/החלפת
> כתובת בלוח של Render, בלי לגעת בקוד.

```
npm install
ADMIN_PASSWORD=test SESSION_SECRET=0123456789abcdef npm start
```
ואז לפתוח `http://localhost:3000`. (בלי טוקן GitHub – הנתונים נשמרים מקומית בתיקיית `data`.)
