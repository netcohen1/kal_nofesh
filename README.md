# קל נופש

אתר מתחמי נופש דינאמי. **שרת Express אחד** (`server.js`) מגיש את החזית ומספק את ה-API,
והנתונים נשמרים בקבצים – בריפו GitHub בפרודקשן (חינם), או על הדיסק המקומי בפיתוח.

**להעלאה לאינטרנט: ראו את `DEPLOY.md` – מדריך פשוט שלב-אחר-שלב (GitHub + Render).**

ההתחברות מאומתת מול סיסמה במשתנה סביבה (מנהל ראשי `admin`) או מול משתמשים שמורים
(מנהלים נוספים). יש גם **מצב פיתוח מקומי** (פתיחת `index.html` ישירות) שעובד מול
localStorage עם סיסמה `demo` – נוח לבדיקת עיצוב בלי שרת.

> תיקיית `netlify/` נשארה כגיבוי בלבד; הפריסה הנוכחית משתמשת ב-`server.js`
> (שמשתמש מחדש בלוגיקת `netlify/functions/_lib/auth.js` ו-`validate.js`).

## בדיקה מקומית (לפני העלאה)

### שלב 1: הרצת שרת מקומי
לחיצה כפולה על `start-dev.bat` (או `start-dev.ps1` ב-PowerShell). הוא ינסה Python ואז Node.js ויפתח דפדפן ב-`http://localhost:5173`.

> חשוב לפתוח את האתר דרך שרת ולא בלחיצה כפולה על `index.html` - אחרת הניווט שובר ומציג רשימת קבצים.

### שלב 2: כניסה לאזור הניהול
- ב-`http://localhost:5173/pages/admin-login.html`
- שם משתמש: השאירו ריק (או `admin`)
- סיסמה: **`demo`**

יופיע באנר צהוב שמסביר שאתם במצב פיתוח מקומי. כל מה שתעשו (הוספת דירות, ערים, סוגי נכס, תקופות, מנהלים, סימון ימים תפוסים, וכו') יישמר ב-localStorage של הדפדפן הזה בלבד.

### שלב 3: ניקוי נתוני פיתוח
בקונסול של הדפדפן: `KN.local.reset()` ואז רענון.

## פריסה ל-Netlify

### שלב 1: העלאת התיקייה
גררו את כל התיקייה ל-Netlify (Drag & drop) או חברו דרך Git.

### שלב 2: משתני סביבה
ב-Site configuration → Environment variables, הוסיפו:

| משתנה | חובה | מטרה |
|-------|------|------|
| `ADMIN_PASSWORD` | כן | סיסמת המנהל הראשי (שם משתמש: `admin`). |
| `SESSION_SECRET` | כן | מחרוזת אקראית באורך 32 תווים לפחות. חותמת על טוקני סשן. |

ליצירת `SESSION_SECRET`:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

או ב-PowerShell:
```powershell
-join (1..32 | ForEach-Object { '{0:x2}' -f (Get-Random -Max 256) })
```

### שלב 3: Deploy חוזר
לאחר הגדרת המשתנים, הריצו Deploy מחדש כדי שיתקבלו.

### שלב 4: כניסה ראשונה
- גשו ל-`/pages/admin-login.html`
- שם משתמש ריק, סיסמה = `ADMIN_PASSWORD` שלכם
- הוסיפו סוגי נכס, ערים, תקופות, ולבסוף את הדירה הראשונה
- ניתן להוסיף מנהלים נוספים מתפריט "מנהלים"

## פיצ׳רים מרכזיים

### חיפוש
- **חיפוש מהיר לפי שם** – שורה נפרדת במרכז עם כפתור משלה
- **חיפוש מתקדם** – סוג נכס, עיר, תאריכים, מחיר
- **שלוש שיטות בחירת תאריכים**:
  1. *כל התקופות* – ללא סינון תאריכים
  2. *תאריכים מדויקים* – בחירת טווח מלוח שנה (הדירה תופיע רק אם פנויה בכל הימים)
  3. *גמיש לפי חודש* – בחירת חודש (הדירה תופיע אם יש לפחות יום אחד פנוי בחודש)

### יומן זמינות
לכל דירה יש לוח שנה במצב ניהול:
- לחיצה על יום מסמנת אותו כתפוס
- מציג חודש נוכחי בברירת מחדל, ניווט חודשים
- בדף הציבורי - אותו לוח לקריאה בלבד עם מקרא

### מנהלים
- **מנהל ראשי** (`admin`) – סיסמה ב-env var, לא ניתן לעריכה דרך ה-UI
- **מנהלים נוספים** – CRUD מלא: הוספה, עריכת פרטים, שינוי סיסמה, מחיקה
- כל מנהל מקבל שם משתמש ייחודי
- סיסמאות נשמרות עם hash+salt (scrypt)

### תוכן ניהולי דינאמי
- **רשימות חיפוש**: סוגי נכס, ערים (עם אזור אופציונלי), תקופות
- **הגדרות**: מטבע, גבולות מחיר, מספר ווצאפ ל"פרסום דירה"
- **דירות**: שדות מלאים לפי האפיון + יומן + תמונות

## אבטחה

- אין סודות בקוד הלקוח, רק במשתני סביבה.
- כל פונקציה שמבצעת כתיבה דורשת `Authorization: Bearer <token>` תקף.
- טוקן נחתם עם HMAC-SHA256 על `SESSION_SECRET`, תקף לשבוע.
- סיסמאות משתמשים: scrypt + salt 16-byte לכל אחד.
- ולידציה גם בלקוח (החזר מהיר) וגם בשרת (אמינות).
- תמונות מוגבלות ל-5MB ולסוגי MIME מורשים.
